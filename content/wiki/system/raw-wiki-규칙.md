# raw-wiki-규칙 (vault 헌법)

`/ingest`·`/query`·`/lint`가 런타임에 참조하는 결정 규칙. **추상 표현 금지 — if-else 트리와 수치 표만.**

---

## A. 진입 결정 트리 (4리프)

> [!danger] Q0. 출처 위치 게이트 (가장 먼저, 예외 없음)
> **파일이 `raw/` 안에 있나?**
> - 아니오 (`Zotero/` 등 다른 폴더) → **즉시 ingest 거부.** 위키에 한 글자도 반영하지 않는다. "raw/로 옮겨 달라"고 안내만. (사용자 절대 지시 2026-06-13)
> - 예 → Q1로 진행.
>
> 이 게이트는 도메인 적합성(Q1)보다 **우선**한다. `Zotero/`는 도메인이 맞아도 ingest 금지.

> [!important] Q0.5. 멱등성 가드 (Q0 다음, raw 본문 읽기 **전**)
> **이 raw가 이미 ingest됐나?** — `wiki/log.md`의 `## [..] ingest` 엔트리 + wiki 노트들의 `sources:` frontmatter를 grep해 raw 파일명을 찾는다.
> - **이미 있음** → 기본 **skip**. "이미 [날짜] ingest됨"만 보고하고 **raw 전체를 읽지 않는다**(토큰 절약). 재처리는 사용자가 *명시적으로* 요청했거나 raw가 변경된 경우에만.
> - **처음 봄** → Q1로 진행.
>
> 목적: 같은 raw를 재처리해 토큰을 중복 지불하는 일 방지. 비용 큰 "본문 읽기"보다 **앞에서** 차단한다.

```
자료를 받으면 (Q0·Q0.5 통과 = raw/ 내부 + 미처리 파일만):
├─ Q1. 도메인(전공 수업 / 3D Vision 연구)과 관련 있나?
│   ├─ 아니오 ─────────────────────────────────→ [반려]
│   └─ 예 ↓
├─ Q2. 정제할 핵심 정보 밀도가 충분한가?
│       (요약·개념·엔티티로 박제할 가치가 있나)
│   ├─ 예, 충분 ───────────────────────────────→ [통과]
│   ├─ 주변적/저밀도 (한두 줄짜리 언급) ────────→ [참조]
│   └─ 판단 어려움 / 영역 불명 ─────────────────→ [모호] (§D)
```

- **[통과]**: full ingest. sources 노트 + 영향받는 개념/엔티티 노트 갱신 + index + log.
- **[참조]**: `wiki/index.md`에 1줄 + 최소 stub 노트(있으면). 깊은 정제 안 함.
- **[모호]**: §D 절차. 사용자 선택 전까지 wiki 미반영.
- **[반려]**: wiki에 넣지 않음. `log.md`에 `반려` 사유만 1줄 기록.

## B. 영역 분류 결정 트리 (4분기)

```
[통과]·[참조] 자료의 영역:
├─ ① 학과 수업(강의·과제·시험·교재)에서 나온 자료?
│        → wiki/courses/{과목}/
├─ ② 3D Vision / 3DGS / NeRF / Scene Reconstruction 논문·연구 자료?
│        → wiki/research/
├─ ③ 특정 과목·연구에 종속되지 않는 일반 수학·CS 개념?
│     (예: 선형대수, 군론/Lie group, 확률, 최적화, 해시함수)
│        → wiki/concepts/
└─ ④ 위 어디에도 명확히 안 맞음
         → [모호] (§D, 사용자 개입)
```

> 승격 규칙: 처음엔 ①/②에 두되, 다른 영역에서도 재등장하면 ③ concepts로 옮기고 원위치엔 링크만 남긴다.

## C. 압축 룰

| 항목 | 룰 |
|---|---|
| **읽기 범위** | ingest 대상 raw는 **전체 통독** (선택/표적 읽기 금지 — 누락 방지) |
| 압축률 | raw 1000줄 → wiki 100~200줄 (10~20%) |
| 노트 길이 상한 | 200줄 — 넘으면 분리 |
| 직접 인용 | raw 1자료당 ≤ 3줄 |
| Cross-link 최소 | 3개 — 그중 다른 영역 ≥ 1개 |

> 읽기는 전체, 출력(wiki)은 압축. 둘은 별개다 — 다 읽되 10~20%로 정제한다. 토큰 절약은 Q0.5(이미 처리한 raw skip)로만.
