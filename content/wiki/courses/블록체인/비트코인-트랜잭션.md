# 비트코인 트랜잭션

## 한 줄 정의
> 한 엔터티에서 다른 엔터티로의 **가치 이동**을 표현하는 비트코인의 핵심 단위. (출처: [[2026-06-11-블록체인-강의]])

## 4 구성요소
| 요소 | 직렬화 | 설명 |
|---|---|---|
| 버전 | 4B LE | 사용 가능 기능 규정(보통 1, 일부 2) |
| 입력 | varint+목록 | 쓸 비트코인 정의 |
| 출력 | varint+목록 | 종착지 정의 |
| 록타임 | 4B LE | 유효 시점 |
- Tx ID = 직렬화 후 `hash256`.

## 입력 (이전 출력 참조 + 소유 증명)
4 하부필드: ① 이전 Tx ID(`hash256`) ② 이전 출력 index ③ **해제 스크립트(ScriptSig)**=[[ECDSA-디지털서명|서명]] 포함 ④ 시퀀스.

## 출력
2 하부필드: ① 금액(사토시, 1 BTC=1억 사토시, 8B LE) ② **잠금 스크립트(ScriptPubKey)**([[Script]]). "누구나 넣되 열쇠 소유자만 여는 금고".

## UTXO
- **UTXO 집합** = 미사용 출력 전체 = 현재 유통 중인 모든 비트코인. 풀 노드가 최신 유지·인덱싱 → 검증 가속.
- **이중지불 방지**: 입력이 UTXO 집합에 없는 출력을 가리키면 검증 탈락·전파 안 됨.

## varint (가변 정수)
| 범위 | 표현 | 예 |
|---|---|---|
| 0~252 | 1B | `100`→`0x64` |
| 253~$2^{16}-1$ | `0xfd`+2B LE | `555`→`0xfd2b02` |
| ~$2^{32}-1$ | `0xfe`+4B LE | `70015`→`0xfe7f110100` |
| ~$2^{64}-1$ | `0xff`+8B LE | `18005558675309`→`0xff6dc7ed3e60100000` |

## 록타임
- ≥5억이면 유닉스타임, 미만이면 블록 높이. 도달 전 소비 불가. `OP_CHECKLOCKTIMEVERIFY`(BIP65)로 무효화 공격 방지.
